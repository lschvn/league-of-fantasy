# UC01: Create a Public Game

## Overview
**Goal:** Allow a user to create a game open to all other users.

| Field              | Content                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**             | UC01                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Primary Actor**  | User                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Secondary Actor**| System                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Trigger**        | The user clicks “Create Game” and selects the public mode                                                                                                                                                                                                                                                                                                                                                                                                      |

---

## Description
The user creates a new public game. The game becomes visible in the list of joinable games.

## Conditions

### Preconditions
- The user is authenticated.
- The creation form is accessible.

### Postconditions (Success)
- The public game is created.
- The user is registered as the creator and first participant.
- A fantasy team is associated with the user.

### Postconditions (Failure)
- No game is created.
- No inconsistent business data is recorded.

---

## Scenarios

### Main Scenario
1. The user opens the creation page.
2. The system displays the form.
3. The user selects the “public” type.
4. The user enters the required information.
5. The user submits the form.
6. The system validates the data.
7. The system creates the game.
8. The system adds the user as creator.
9. The system automatically creates the user’s fantasy team.
10. The system makes the game publicly visible.
11. The system displays a confirmation message.

### Alternative Scenarios
- **A1. Some data is invalid:** The system displays errors and requests correction.
- **A2. The game name already exists:** The system refuses the creation.

### Exceptions
- **E1. Technical error during saving:** The system cancels the operation and displays a failure message.

---

## Business Rules
- **BR1.** A public game must appear in the public games list.
- **BR2.** The creator automatically becomes a member of the game.
- **BR3.** An initial fantasy team must be created for the creator.
