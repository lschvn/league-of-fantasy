# UC04: Join and Play a Private Game

## Overview
**Goal:** Allow a user to join a private game using a valid invitation and participate in the gameplay.

## Description
The user uses an invitation code or link to join a private game. Once admitted, the user receives a fantasy team and participates in fantasy gameplay.

## Conditions

### Preconditions
- The user is authenticated.
- The invitation is valid.
- The game still accepts members.

### Postconditions (Success)
- The user joins the private game.
- A fantasy team is associated with the user.
- The user can participate in auctions, set a lineup, and view rankings.

### Postconditions (Failure)
- The user cannot access the game.
- No fantasy team is created.

---

## Scenarios

### Main Scenario
1. The user accesses the private join screen.
2. The user enters the code or uses the received link.
3. The system verifies the game’s existence.
4. The system verifies the validity of the invitation.
5. The system verifies that the user can still join the game.
6. The system records the membership.
7. The system creates the user’s fantasy team.
8. The user accesses the private game space.
9. The user participates in auctions.
10. The user manages their roster.
11. The user sets their lineup.
12. The system updates scores and rankings.

### Alternative Scenarios
- **A1. The code is invalid:** The system refuses access.
- **A2. The code has expired:** The system refuses access.
- **A3. The game is full:** The system refuses entry.
- **A4. The user is already a member:** The system redirects the user to the existing game.

### Exceptions
- **E1. Technical error during fantasy team creation:** The membership is cancelled to avoid a partially valid state.

---

## Business Rules
- **BR1.** A private invitation must be unique, verifiable, and controlled.
- **BR2.** A user can join the same private game only once.
- **BR3.** Each member has a unique fantasy team within the game.
